# KIGO Conventions · KIGO 표준 스타일 가이드 요약

Source: **KIGO 표준 스타일 가이드(한국어) v2.0** (2015-11-03), 한국IT산업세계화학회(KIGO).
IT 업계 공통 영한 번역 표준이며 ko.javascript.info 번역 모범 사례의 원전이다.
이 문서는 원문을 재배포하지 않고 규칙을 요약·재서술한 것이다. 원문 문의는
[KIGO 페이스북 그룹](https://www.facebook.com/groups/kigostyleguide/) 참고.

쌍점·괄호·약어 순서·특수 기호 명칭 등 wiki와 겹치는 규칙은 `formatting.md`,
`terminology.md`에 이미 있다. 아래는 KIGO에만 있는 규칙이다.

## 시제 · Tense

미래 시제는 현재형으로 옮긴다(lint: KO-W010).

- "You will find it easier to…" → "…하는 것이 더 쉽습니다" ("쉬울 것입니다" ✕)
- "We will use infrastructure services." → "인프라 서비스를 사용합니다."

## 조사 붙이기 · Particles

- **괄호 병기 뒤 조사는 괄호 앞 단어에 맞춘다.** 첨부 파일(계약서)을 (○) /
  첨부 파일(계약서)를 (✕). SMTP(Simple Mail Transfer Protocol)를 (약어 SMTP 기준).
- **함수 표현식은 예외로 괄호 내용까지 읽고 조사를 붙인다.** `pot(A,B)`는 (○) /
  `pot(A,B)`은 (✕). 함수와 괄호를 순차적으로 모두 읽기 때문이다.
- **플레이스홀더 변수** 뒤엔 이중 조사 '은(는)', '이(가)', '을(를)'을 쓰거나
  이중 조사가 필요 없는 문형으로 바꾼다("친구 {%c} 님이 왔습니다").
- **숫자·외래어 뒤 조사는 소리 나는 대로** 붙인다. 45는, USB 4.0은, setup.exe를,
  ASCII를, RAM을, DNA는.

## 약어 · Abbreviations

- 한국어에선 약어를 잘 쓰지 않으므로 원래 발음이나 번역으로 푼다. 10 ft. → 10피트,
  etc. → 등, Mon~Sun → 월~일.
- 약어는 원문이 소문자라도 대문자로: Ftp → FTP. 복수형 s는 떼어낸다: PDAs → PDA.
- IT 분야는 약어를 앞에 두고 풀이를 괄호로: USB(범용직렬버스). IT 외 분야는 한글
  번역 뒤 괄호: 미국중재협회(AAA, American Arbitration Association).

## 외래어 표기 세칙 · Transliteration details

국립국어원 표기법의 실무 요점이다.

- 받침엔 ㄱ·ㄴ·ㄹ·ㅁ·ㅂ·ㅅ·ㅇ만 쓴다. 파열음에 된소리를 쓰지 않는다.
- 단모음 뒤 어말 무성 파열음(p·t·k)은 받침으로: 컵, 캣, 인터넷, 서브셋, 템플릿, 툴킷.
  단모음이 아니면 '으'를 붙인다: 케이크, 테이프. 관용 예외: 세트, 네트, 키트.
- ㅈ·ㅊ 뒤엔 단모음으로: 차트(챠트 ✕), 버전(버젼 ✕), 프로시저(프로시져 ✕).
- 장음은 표기하지 않는다: 팀(티임 ✕). ou는 '오': 새도(shadow).
- 지명은 원지음: 말레이시아, 베네수엘라, 파리.

## 문장·어법 · Sentence-level

- **경어체가 기본.** 높임 '-시-'는 한 문장에서 맨 끝 동사에만: "구매할 때에는
  사용하시기 바랍니다" ("구매하실 때에는 사용하시기" ✕).
- **단계별 사용법의 명령형은 평서형으로 부드럽게**: "1. 속성을 선택합니다."
  ("선택하십시오" 보다 권장).
- **주체가 분명하면 생략하되, 한 문장에 서로 다른 주체가 있으면 명확히 구별한다.**
- **격조사 '~의'를 연이어 쓰지 않는다**(lint: KO-W009). "당사의 제품의 버전" →
  "당사 제품 버전". 불필요한 조사를 걷어내고 말을 풀어 간단하게 쓴다.
- 두 문장 사이의 세미콜론(;)은 쉼표나 마침표로 바꾼다. 문장 끝 콜론은 마침표로,
  구 형태 라벨의 콜론은 유지한다("작성 날짜:").
- 자주 틀리는 말(lint: KO-E006): 예/아니오→예/아니요, 그리고 나서→그러고 나서,
  몇 일→며칠, 하므로써→함으로써, 삼가하다→삼가다, 잠궈→잠가.

## 숫자·단위·날짜 · Numbers, units, dates

- 숫자는 천 단위 쉼표: 1,000,000. 단 해상도 픽셀 수는 예외(1280 x 1024).
  한글로 읽을 땐 만 단위로 묶는다(1억 2345만 6700).
- **숫자와 단위는 붙여 쓴다**(IT 분야, lint: KO-W011): 2,450kg, 10/100Mbps.
  측정 단위는 소문자로 표기한다.
- 단위 명사는 띄어 쓴다(열 살, 집 한 채). 단 순서·숫자와 함께면 붙일 수 있다
  (5개의 백업, 설명서 1권).
- 두 개 이상의 숫자엔 단위를 반복한다: "32비트 및 64비트 Windows 운영체제".
- 날짜는 '연/월/일/시/분/초' 또는 yyyy-mm-dd hh:mm:ss. 통화는 ₩10,000,
  외화는 "미화 10,000달러". 백분율은 10%(공백 없이).

## 소프트웨어 현지화 · Software localization

skill 기존 문서가 다루지 않는 UI·매뉴얼 번역 영역이다.

- **제품·회사·운영체제·소프트웨어 이름은 번역하지 않는다.** 상표(™·®)가 붙은
  단어는 특히 주의한다. 저작권·상표 고지문은 번역하지 않는다(©가 지원되지 않는
  환경에선 (c), (R), (TM)으로 대체).
- **UI 문자열은 출시된 제품의 실제 UI를 따른다.** 현지화된 제품은 한국어 UI 명칭
  그대로("시작 -> 모든 프로그램"), 현지화되지 않은 제품은 '영문 UI(한국어 번역)'
  형식("'Edit Parameter information(매개변수 정보 편집)' 단추").
- **UI 요소는 굵은 글꼴이나 대괄호로 구분한다**: "**추가**를 클릭합니다" 또는
  "[추가]를 클릭합니다". 이탤릭은 한글 가독성이 나빠 강조로 쓰지 않는다(작은따옴표나
  굵게로 대체).
- **자판 키는 자판에 표시된 그대로**: Ctrl+Shift(ctrl+shift ✕), iMac은 command.
- **선택 키(&)는 번역된 메뉴 이름 뒤 괄호로**: &Next → 다음(&N). 소문자 선택 키는
  대문자로 바꾼다(보기(&W)). 특수 기호로 끝나면 기호 앞에(파일(&F):).
- **매개변수·플레이스홀더는 일관성 있게**: 번역하지 않는 것이 기본이나 번역한다면
  전체 작업에서 통일한다. `\n`·`:` 주변 공백은 원문을 따른다.
- **매뉴얼 제목은 명사형으로**: "Testing a setting" → "설정 테스트"
  ("설정을 테스트합니다" ✕). 단 만들기·보기 같은 고유어는 '-하기' 허용("새 파일
  만들기").
- 위치 표현: Middle(종적)은 '중간', Center(횡적)는 '가운데'.
- 정렬 순서: 기호 > 숫자 > 한글(자모순) > 영문(알파벳순).

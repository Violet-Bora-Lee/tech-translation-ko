# 번역 교정 diff 코딩 결과

이보라가 다른 사람의 기존 번역을 교정한 커밋 135개(`docs/analysis/review_commits.json`)의 마크다운 diff를 헝크(hunk) 단위로 읽고 다섯 범주로 제한 코딩한 결과다. 분류는 변경된 텍스트 자체만 근거로 삼았고, 왜 그렇게 고쳤을지 의도는 추론하지 않았다.

## 방법

135개 커밋 각각을 `git show <hash> -- '*.md'`로 추출해 diff 파일로 저장한 뒤 다섯 개 청크(27개씩)로 나누어 병렬로 분류했다. 청크 하나(`115_d80e6967`)는 en.javascript.info 업스트림과의 대규모 merge 커밋으로 158개 파일, 397개 헝크를 포함하고 있어 전체 헝크 수의 상당 부분을 차지한다. 이 커밋은 여러 기여자의 변경이 뒤섞여 있어 순수한 "이보라의 교정"으로 보기 어려운 헝크가 많으므로, 아래 집계에서 이 점을 감안해서 읽어야 한다.

총 1,155개 헝크를 분류했다.

## 범주별 빈도

| 범주 | 헝크 수 | 비율 |
|---|---:|---:|
| 원문 갱신 대응 | 336 | 29.1% |
| 문체 통일 | 300 | 26.0% |
| 판정 불가 | 302 | 26.1% |
| 의미 오류 수정 | 147 | 12.7% |
| 용어 변경 | 70 | 6.1% |

**원문 갱신 대응**이 가장 큰 비중을 차지하지만 이는 앞서 언급한 대형 merge 커밋(`d80e6967`) 하나가 끌어올린 수치다. 이 커밋을 제외한 134개 커밋만 놓고 보면 **문체 통일**과 **판정 불가**가 가장 큰 비중을 차지하고, 그다음이 **의미 오류 수정**, **용어 변경** 순이다.

**판정 불가**(302건, 26.1%)의 상당수는 다섯 범주 중 어디에도 깔끔히 들어맞지 않는 두 가지 패턴이었다. 첫째, 기존 번역을 교정한 게 아니라 영어 원문이 그대로 남아 있던 과제·해답 문서를 이보라가 처음으로 한국어로 옮긴 헝크(신규 번역이지 교정이 아님). 둘째, README·CONTRIBUTING 같은 메타 문서의 링크 추가나 사소한 마크다운 포맷 변경. 두 경우 모두 "타인의 번역을 교정"이라는 원래 정의에서 벗어나 있어 판정 불가로 남겨두었다.

## 반복 수정 유형 순위

같은 유형의 수정이 한 커밋 안에서 반복되는 경우가 많았다. 관찰된 반복 유형을 빈도 순으로 정리하면 다음과 같다(대략적인 관찰 횟수).

1. **오탈자·맞춤법 일괄 교정** — "대게"→"대개"(같은 커밋 내 약 32회), "제너레티어"→"제너레이터", "쳅터"→"챕터" 등. 맞춤법 점검성 커밋 한 건에서 수십 건씩 나온다.
2. **강조용 큰따옴표 → 작은따옴표 통일** — 커밋 하나에서 약 20회 관찰. CLAUDE.md 한국어 글쓰기 규칙(강조는 작은따옴표)과 일치한다.
3. **문장 끝 쌍점(:) 제거** — "예를 들어:"→"예시" 류. 여러 커밋에 걸쳐 반복 관찰.
4. **헤딩에서 따옴표·영어 병기 제거** — `## "use strict"`→`## use strict`, `## 'get' 트랩`→`## get 트랩`(트랩 헤딩만 한 커밋에서 약 6회), `*숫자형(Number)*`→`*숫자형*` 류 보조 개념어 병기 삭제.
5. **용어 일괄 치환(커밋 메시지에 명시)** — "기초"→"기본"(문서 전반, 약 12회), "spread syntax"→"전개 구문"(여러 파일에 걸쳐 약 15회) 같이 커밋 메시지 자체에 `[용어 변경]` 또는 `[용어 통일]`이라고 밝힌 일괄 교체.
6. **인칭대명사 삭제** — "우리는" 등 we/you 계열 삭제. 한 커밋 내 반복 관찰.
7. **짝 단어 표기를 가운뎃점(·)으로 통일** — "대소문자"·"대/소문자"→"대·소문자"(같은 커밋 내 약 9회).
8. **P.S. 라벨을 하우스 스타일로 통일** — "P.S. ~"를 "더 생각해보기 N:" 형태로 교체(약 4회).
9. **예제 속 에러 메시지·인명을 한국어로 현지화** — `throw new Error("Whoops!")`→`throw new Error("에러 발생!")`(약 10회), `"John"`→`"보라"` 등.
10. **자료형 표기 통일** — "숫자` + `형"처럼 백틱과 분리 표기하던 것을 "숫자형" 통합 표기로(문자형·불린형·객체형·심볼형에도 동일 적용, 약 5회).

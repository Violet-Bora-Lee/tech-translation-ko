# 운영 스프레드시트 암묵지 정리

> `glossary.tsv`에는 이후 '2025 AI 동향과 이슈로 살펴보는 AI 시대에 꼭 알아야 할
> 핵심용어' 책자(배포용 PDF)에서 추출한 AI 용어 169개가 추가됐다. 해당 항목은
> note에 '(2025 AI 핵심용어)' 표기가 있다.

출처는 ko.javascript.info 운영 스프레드시트(JavaScript Tutorial - Task Manager)이다. 진행 관리 시트(전체 대시보드, 본문·과제 번역 현황, 1·2차 리뷰 현황, 기여자)는 제외하고 용어집(보호), 용어집(편집가능), 참고 자료, 논의사항 시트를 수집했다. 용어 데이터는 `glossary.tsv` 참고.

## 자주 하는 실수 (참고 자료 시트)

- chapter는 '챕터'로 번역한다. 초기엔 '주제'였으나 합의로 변경.
- property는 '프로퍼티'. '속성'은 attribute의 번역어다.
- 새롭게 등장하는 키워드는 한-영 병기한다. 예: 프로퍼티(property), 브라우저 객체 모델(Browser Object Model, BOM). 영-한 병기가 아니라 한-영 병기다.
- 원문의 we, you 등 인칭대명사는 되도록 해석하지 않는다.
- 대명사·지시어는 되도록 풀어쓴다. 원문에선 태·수로 지시 대상을 파악할 수 있으나 번역문에선 힘들다.

## 논의사항 시트에서 드러난 의사결정 패턴

용어 결정 과정 기록이다. 결정 자체보다 '어떻게 결정하는가'가 암묵지다.

1. **syntax → 문법**: MDN 표기를 근거로 논의 후 확정. 권위 있는 한국어 문서(MDN)를 1차 근거로 삼는다.
2. **chapter → 챕터**: 여러 번역자가 혼용하자 통일을 위해 합의. 공동 작업에선 통일이 정확성만큼 중요하다.
3. **primitive → 원시 값, primitive type → 원시 타입**: 출판된 도서(모던 자바스크립트) 표기를 따름. 최초 번역자와 협의 후 확정.
4. **boolean → 불(린)값**: 출판사 권유('불 값')와 러닝 자바스크립트('불리언 값')가 갈리자, 한글 번역은 필수로 하되 번역어는 문맥에 따라 혼용하기로 결정. 상충하는 권위 사이에선 '번역 여부'만 고정하고 표기는 유연하게 둔다.
5. **construct → 문법 (구조), 구문**: 확신이 없으면 잠정 등록 후 의견을 청한다. 이후 MS 용어집을 참고해 '구문' 혼용으로 갱신.
6. **어려운 문장은 공개 질의**: 해석이 안 되는 원문을 논의 시트에 올려 협업으로 푼다. 제안된 번역의 주술 구조가 맞지 않으면 재요청한다("주술구조가 안 맞는 문장 같습니다. 조금 더 풀어주실 수 있을까요?").
7. **custom → 커스텀/사용자 지정 혼용**: MS Terminology 근거로 '사용자 지정'을 제안했다가, 실제 문장에 적용해 보니 의미가 안 와닿는 경우가 많아 혼용으로 결정. 근거보다 실제 문장 적합성을 우선한다.
8. **용어 변경 이력을 남긴다**: 번역어를 바꿀 때 '기존 X에서 Y로 변경합니다'와 근거를 용어집에 기록한다.

## 번역어 선정 기준(우선순위)

논의·용어집 노트에서 반복적으로 인용되는 근거의 순서.

1. 출판된 도서(모던 자바스크립트 Deep Dive, 러닝 자바스크립트, 프론트엔드 개발자를 위한 자바스크립트 프로그래밍, 인사이드 자바스크립트, 초보자를 위한 JavaScript 200제 등)
2. MDN 한국어 문서
3. 마이크로소프트 Language Portal Terminology Search
4. 국립국어원 외래어 표기법 용례(스로틀, 섬네일, 다이내믹, 아키텍처 등)
5. 한글라이즈(음차 표기)
6. 위 근거가 갈리면 팀 논의로 결정하고 이력을 남긴다

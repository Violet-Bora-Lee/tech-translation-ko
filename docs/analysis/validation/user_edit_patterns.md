# SSE 번역 수기 수정 diff에서 추출한 패턴

2026-07-19, server-sent-events 번역(skill 산출물)을 번역자가 직접 퇴고한 diff 분석이다.
12개 헝크에서 추출했고 sentence-patterns.md에 반영했다.

## 추출 패턴

1. **동작 주체와 자동성을 명시한다.** "브라우저는 접속해" → "브라우저가 알아서 접속해"
   (3회), "브라우저가 요청을 보낼 때 자동으로 붙인 `Origin` 헤더", "기본값(브라우저에서
   지정)". 개발자가 하는 일과 브라우저·서버가 자동으로 하는 일의 경계를 문장에 드러낸다.
2. **클라이언트·서버 아키텍처 맥락을 복원한다.** "메시지를 받으려면 ~ 만들기만 하면
   됩니다" → "서버에서 보내는 메시지는 브라우저에서 실행 중인 클라이언트 측
   자바스크립트에서 ~". 코드가 어느 쪽에서 실행되는지 명시. "메시지를 써 내려갑니다" →
   "메시지를 클라이언트로 흘려보냅니다"(방향 명시).
3. **일반 동작과의 대비를 보강한다.** "상태 코드 200과 헤더로 응답한 뒤" → "본문 전체를
   보내는 대신 우선 상태 코드 200과 헤더 ~만 보내 응답한 뒤". 일반 HTTP 응답과 뭐가
   다른지 '대신·우선·만'으로 드러낸다.
4. **프로토콜 흐름은 요청-응답 단계 문장으로 분해한다.** Origin/CORS 한 문장을
   '서버는 ~헤더를 받습니다. 요청을 받은 서버는 ~허용한다고 응답합니다.' 두 단계로.
5. **주제문을 세우고 상세를 잇는다.** "서버는 응답에 `retry:`를 실어 ~" 앞에
   "재연결 지연 시간은 서버가 원하는 값으로 바꿀 수 있습니다."를 신설.
6. **전환 표지로 앞 내용을 요약해 받는다.** "다만," 추가. "이렇게 서버나 브라우저에서
   명시적으로 재연결을 멈출 때 말고," (앞 리스트를 요약해 문두에 복원).
7. **헤딩은 'X로 ~하기'로 통일한다.** "커넥션 상태 확인하기, readyState" →
   "readyState로 커넥션 상태 확인하기". 쉼표 병기형보다 도구 중심 동작형.
8. **손에 잡히는 명사로 구체화한다.** "시세" → "주식 시세".
9. **단정을 완화한다.** "능력이 부족한 수단입니다" → "부족한 수단 같아 보입니다"
   (비교표만 보고 내린 인상임을 표현).
10. **지시어 대신 명사 반복으로 인과를 잇는다.** "줄바꿈이 ~ 인코딩되기 때문에" →
    "JSON으로 인코딩하면 줄바꿈이 ~ 처리되기 때문에" (앞 문장 명사를 받아 조건 명시).

## 수기 수정에서 발견된 맞춤법 실수(검사기 필요성 재확인)

- 의존 명사 '때' 붙여쓰기: 보낼때·멈출때 (→ 린터 KO-W012 추가)
- '중' 붙여쓰기: 실행중인 (→ 린터 KO-W013 추가)
- '같아' 붙여쓰기: 수단같아, '인코딩 하면' 띄어쓰기, '업니다'(엽니다 오타)

# mutation-observer 번역 수기 수정 diff에서 추출한 패턴

2026-07-19, mutation-observer 번역(skill 산출물) 퇴고 diff 분석이다. 18개 헝크에서
추출했고 기존 패턴을 확증하는 사례는 번호를 붙여 참조했다.

## 추출 패턴

11. **이중 주제 구문을 명사구로 푼다.** "`MutationObserver`는 사용법이 간단합니다" →
    "`MutationObserver` 사용법은 간단합니다". 'X는 Y가 ~하다'의 겹주어를
    'X Y는 ~하다'로 붙여 한 주어로 만든다.
12. **존재문을 행위문으로 바꾼다.** "스크립트엔 제거할 수단이 없습니다" →
    "스크립트는 제거할 수단을 제공하지 않습니다". '~에 ~이 있다/없다'보다 행위
    주체를 세우고 동사로 서술한다. "메서드가 있습니다" → "메서드도 존재합니다"처럼
    조사('도')로 앞 문맥과의 관계도 함께 드러낸다.
13. **실행 방법을 화면 기준으로 안내한다.** "위 코드를 브라우저에서 실행하고" →
    "우측 상단 재생 버튼을 눌러 코드를 브라우저에서 실행하고". 출력 위치도 실제
    화면으로: "`console.log`에 출력됩니다" → "`console.log` 때문에 개발자 도구에
    출력됩니다". 튜토리얼 플랫폼의 실제 UI(재생 버튼, 개발자 도구)를 확인해 쓴다
    (SSE 패턴 아님, Viki 문체 패턴 22 '실제 화면과 어긋난 서술 교정'의 번역판).
14. **가정·추정임을 어미로 명시한다.** "라이브러리를 사용해 보겠습니다" →
    "사용해 본다고 가정하겠습니다"(가상 시나리오), "덧붙일 수 있습니다" →
    "덧붙일 수 있을 겁니다"(뒤에 반전이 올 잠정 해법). SSE 패턴 9(단정 완화)의 확장.
15. **소속·위치어를 붙여 지시를 좁힌다.** "첫 번째 인수로" → "콜백 함수의 첫 번째
    인수로", "호출을" → "위와 같은 코드를", "아래 코드" → "아래쪽 코드",
    "밑에 있는 요소" → "하위 요소"(DOM 계층 용어로). SSE 패턴 8·10 계열.
16. **코드 주석은 수사 의문 대신 코드의 행동을 서술한다.** "새 노드를 검사합니다.
    강조 표시할 대상이 있을까요?" → "강조 표시할 새 노드가 있는지 검사합니다.",
    "있을 수도 있습니다" → "있을 수 있으니 확인합니다"(다음 줄이 하는 일로 연결).
    주석 속 지시 대상은 실제 식별자로: "호출해야 합니다" → "takeRecords를 호출해야
    합니다", "요소만" → "요소(노드)만".
17. **요약부는 달성 상태와 효용으로 맺는다.** "전부 추적합니다" → "전부 추적할 수
    있게 되었습니다", "추가하거나 삭제할 수 있게 되었죠" → "추가되거나 삭제되어도
    알아서 강조 표시가 됩니다"('알아서' 자동성, SSE 패턴 1 재확인).
18. **담화 연결어를 신설한다.** 해법 문장 앞에 "이럴 때", 질문 앞에 "그럼". 문두
    "..." 같은 영어식 부호는 삭제(formatting 규칙 재확인). SSE 패턴 6 계열.

# 번역 패턴 코드북 v1

en↔ko 대조 분석에서 패턴을 분류하는 기준이다. 보고 시 아래 카테고리 ID를 사용한다.

## 카테고리

- **A. 문장 분해·재구성**: 긴 복문을 짧은 문장 여러 개로 나누거나, 두 문장을 합치거나, 평서문↔의문문 전환 등 문장 단위 구조 변경
- **B. 어순·정보구조 재배치**: 원문 어순을 버리고 한국어 정보 흐름(주제 먼저, 결론 뒤)으로 재배치. 열거 구조 명시화("First," → "첫 번째 제약은 ~입니다") 포함
- **C. 인칭·대명사 처리**: we/you/it/this 생략 또는 지시 대상 명시로 풀어쓰기
- **D. 어휘 선택·용어**: 용어집 적용, 한-영 병기, 음차 결정, 같은 단어의 유의어 교체(단조로움 회피), 관용구 대응
- **E. 어미·문체**: 합니다체, '-죠' 같은 리듬 어미, 청유형('~해봅시다'), '-시-' 제한, 피동 회피
- **F. 문장부호·표기**: 쌍점 제거, 큰따옴표→작은따옴표, 가운뎃점, 괄호와 마침표 위치, 명사 종결 무마침표, 헤딩 문장부호 제거
- **G. 코드 요소 처리**: 주석 번역 방식(설명 보강 포함), 코드 내 문자열·UI 텍스트 번역, 에러 메시지 원문 유지, smart header(```smart header="...") 처리, 이미지 캡션
- **H. 생략·보강**: 원문에 없는 설명 추가('- 옮긴이' 주 포함), 원문 중복·군더더기 생략, 독자 배려 보강
- **I. 담화 연결**: 문장 사이 인과·대조를 접속어로 명시(그럼에도 불구하고, ~기 때문입니다), 문단 도입부 연결
- **J. 신규 패턴**: 위에 없는 발견. 자유 기술

## 보고 형식

파일마다 발견한 패턴을 아래 형식으로 기록한다.

```
### [카테고리ID] 패턴 한 줄 요약
- 원문: (en 문장)
- 번역: (ko 문장)
- 위치: 파일경로:줄번호
- 설명: 무엇을 어떻게 바꿨고 왜 효과적인지 1~2문장
```

같은 패턴이 반복되면 대표 사례 1개 + 빈도(몇 회 관찰)만 기록한다. 우선순위는 '기계적으로 규칙화하기 어려운 암묵지 > 이미 wiki에 명문화된 규칙 확인'이다. wiki 명문 규칙(쌍점, 인칭대명사, 단수형 등)과 일치하는 사례는 빈도만 세고, 명문화되지 않은 기법은 상세히 기록한다.
